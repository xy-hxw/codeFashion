访问者模式

    1：将行为集中到一个访问者对象中，其改变不影响系统数据结构


    优点
        1：访问者模式符合单一职责原则
        2：元素类中封装在访问者中的操作是与元素类本身关系不大且易改变的操作
        3：被封装的操作发生改变时可以在不改变元素类的前提下，实现对变化部分的扩展
        4：接受不同的访问者来实现不同的操作
        5：存在相似的操作，避免大量重复代码，可以将重复代码封装到访问者中


    缺点
        1：增加新的元素类需要修改访问者类和访问者类的实现类
        2：访问者模式适用于已有功能的重构

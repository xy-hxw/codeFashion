迭代子模式

    jdk 中 iterator
        JAVA语言以接口java.util.Iterator的方式支持迭代子模式

    1：又名游标模式，可以顺序访问一个聚集中的元素而不暴露聚集的内部
    2：为不同的聚集提供相同的遍历方式，使得客户端只需要关注迭代接口，不需要关注所迭代的聚集的具体结构
    3：两种实现方式 ：白箱聚集与外禀迭代子、黑箱聚集与内禀迭代子

    白箱聚集
        1：白箱聚集提供了迭代方法，客户端可以自行进行迭代。但是迭代子对象和迭代模式会将迭代过程抽象化，
            将客户端和迭代子的责任分离开。
        2：聚集对象的种类发送变化，或者迭代的方法发生变化。迭代子作为中间层可以吸收变化的因素，避免
            修改客户端或者聚集本身

    黑箱聚集
        1：聚集的接口没有提供修改聚集元素的方法
        2：聚集对象的内部结构对迭代子对象公开
        3：将迭代子类设计成聚集内的内部成员类，迭代子对象可以像聚集对象的内部成员一样访问聚集对象的内部结构

    优点：
        1：迭代子模式简化了聚集的接口
        2：系统需要针对几个不同的聚集对象进行迭代，而这些对象提供的遍历方法有所不同的时，使用迭代子模式和外界的
            迭代子对象是有意义的
        3：系统可以使用一个统一的接口进行所有的迭代
        4：每一个聚集对象可以有多个迭代子对象，每一个迭代子的状态是彼此独立的（Fail Fast机制），一个对象可以同时
            有几个迭代在进行之中
        5：遍历算法封装在迭代子中，迭代的算法可以独立于聚集角色的变化

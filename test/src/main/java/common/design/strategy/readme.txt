策略模式

    概念：有一组算法或策略，我们可以根据不同的场景选择不同的算法或策略

    优点：
        1：动态的改变对象的行为

    缺点：
        1：客户端必须知道所有的策略类，并决定使用哪一个策略类
        2：会产生很多策略类

    Example
        TreeSet 构造方法需要一个比较器
        Set<String> set = new TreeSet<>(String::compareTo);

    组成：
        1：抽象策略角色，比如上面的Comparator
        2：具体策略角色，实现Comparator接口的一组实现类
        3：环境角色，内部持有一个抽象角色的引用，给客户端调用
